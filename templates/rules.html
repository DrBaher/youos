Rules

Filter→action rules. When a sweep fetches a message, every rule whose conditions all match shapes the draft (skip / decline / prepend / hold) or routes the message (label / archive / star / mark read / mark important). Routing is gated behind Settings → agent actions and dry-run by default.

Add a rule

⚠ Forwarding sends mail and is irreversible. A forward rule saves fine, but it won’t actually send until you enable agent.send.enabled + agent.actions.allow_forward (and routing is live, kill-switch off) in Settings. Until then it records as blocked.
